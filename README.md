# ProqnozAI — Telegram-бот AI-прогнозов на спорт

## О проекте

ProqnozAI — Telegram-бот на базе Anthropic Claude, который даёт пользователям
AI-прогнозы на спортивные матчи (футбол, UFC/MMA, баскетбол, теннис, хоккей).
Целевой рынок — Азербайджан, Турция, Казахстан, Узбекистан, арабские страны
и русскоязычная аудитория.

Бот **не является букмекером** — он только аналитика. Реальные коэффициенты
подтягиваются с Mostbet через партнёрский Odds Checker API (IP whitelisted).

## Стек

- **Язык:** Python 3.13
- **Telegram:** `python-telegram-bot==21.5`
- **AI:** Anthropic Claude API (`anthropic` SDK)
  - `claude-sonnet-4-6` — основные прогнозы (расширенный + краткий формат)
  - `claude-haiku-4-5-20251001` — быстрые задачи (live-советы, экспресс, сравнение, парсинг запроса)
- **БД:** SQLite (`bot.db`, файл создаётся автоматически при первом запуске)
- **HTTP-клиент:** `httpx` (async)
- **Хостинг:** Railway Pro (со статичным outbound IP)
- **Репозиторий:** GitHub, приватный

## Переменные окружения (Railway → Variables)

| Переменная | Назначение | Обязательна |
|---|---|---|
| `TELEGRAM_TOKEN` | Токен бота от BotFather | Да |
| `ANTHROPIC_API_KEY` | Ключ Anthropic API | Да |
| `ADMIN_ID` | Telegram ID администратора (доступ к `/admin`) | Да |
| `FOOTBALL_API_KEY` | football-data.org — историческая статистика команд | Нет |
| `APIFOOTBALL_KEY` | api-football.com (api-sports.io) — live-данные матчей, события | Нет |
| `WEBHOOK_URL` | Если задан — бот работает через webhook вместо polling | Нет |
| `PORT` | Порт для webhook-режима (по умолчанию 8080) | Нет |

Без `APIFOOTBALL_KEY` не работают: live-уведомления, кнопка "Следить за матчем".
Без `FOOTBALL_API_KEY` не подтягивается историческая статистика по командам.
Mostbet API не требует отдельного ключа — доступ даётся по IP whitelist
(см. раздел "Mostbet Odds Checker API" ниже).

| `db.py` | `db_init()` создаёт таблицы SQLite, дальше CRUD-функции с префиксом `db_` (юзеры, история, избранное, conversation memory, live-подписки, odds-алерты) |

| `mostbet.py` | Mostbet Odds Checker API: загрузка/кэш списка матчей, поиск матча по названиям команд, получение реальных коэффициентов (см. ниже) |

## Ключевая бизнес-логика

### Регистрация и онбординг
1. `/start` → выбор языка из 7 (определяется автоматически по `language_code` Telegram, юзер может сменить)
2. Ввод имени (без верификации телефона — намеренно убрана)
3. Онбординг — 2 вопроса: вид спорта, опыт в ставках (используется для персонализации промпта)

### Прогноз на матч
1. Пользователь пишет текст или шлёт фото
2. `parse_match_query()` (Claude-haiku) вычленяет команды/дату/спорт
3. Если дата матча > 7 дней вперёд — бот отказывает (`match_too_far`), чтобы не давать прогнозы на матчи без актуальных коэффициентов
4. Бот предлагает выбрать формат: **Расширенный** (полный анализ, все рынки) или **Короткий** (только лучшая ставка)
5. Перед запросом к Claude бот пытается найти матч в кэше Mostbet (`mostbet_find_match`) и получить реальные коэффициенты (`mostbet_get_odds`) — если находит, передаёт их в промпт с инструкцией использовать **только** эти цифры
6. Ответ Claude сохраняется в историю (`db_save_history`) и в контекст разговора (`db_save_conv`)
7. Под прогнозом — кнопки: "Следить за матчем" (если есть live-данные), "В избранное"

### Mostbet Odds Checker API
- Базовый URL: `https://mostbet2.com`
- Эндпоинты:
  - `GET /api/v3/advertiser/oddschecker/line/list` — список матчей (пагинация через `lastId`)
  - `GET /api/v3/advertiser/oddschecker/line/{lineId}/outcomes/list` — коэффициенты по матчу
- Доступ только по whitelisted IP (контакт по доступу: Armen Gulyan, a.gulyan@double.global)
- Список матчей кэшируется в памяти (`mostbet_cache`) на `MOSTBET_CACHE_TTL` (900 сек), обновляется фоновой задачей каждые 15 минут — это сделано из-за агрессивного rate-limit (429) на стороне Mostbet
- `_is_within_week()` фильтрует матчи дальше 7 дней
- Команда `/testapi` (только админ) — прямой тест запроса к Mostbet, показывает статус и первые матчи

### Live-уведомления
- Кнопка "Следить за матчем" появляется только если найден матч через `APIFOOTBALL_KEY`
- `poller()` каждые 60 сек проверяет статус, шлёт: гол, карточка, перерыв, конец матча
- Раз в ~15 минут — случайный FOMO-алерт (ожидается гол / live value / давление)
- При подписке регистрируется алерт на изменение коэффициента (`odds_alerts` таблица)

### Дополнительные фичи
- **Избранное** (`/fav`) — список команд, кнопка добавления под каждым прогнозом
- **История** (`/history`) — последние 5 прогнозов, кнопки фидбека ✅/❌ (сыграло/нет) → копится статистика точности в профиле
- **Экспресс** (`/express`) — выбор 2–5 матчей, Claude собирает ставки + итоговый коэффициент, использует реальные матчи из кэша Mostbet если есть
- **Сравнение** (`/compare`) — сравнение двух команд: форма, очные встречи, xG

### Админ-панель (`/admin`, только `ADMIN_ID`)
- Статистика (пользователи, запросы, языки, онбординг, live-подписки)
- Рассылка всем / по конкретному языку (гео-таргетинг)
- Список заблокированных + разблокировка
- Поиск пользователя (по ID/username/имени) + блок/разблок
- Изменение языка пользователя вручную
- Live-подписки (сколько активных)
- Тест Mostbet API

## Известные особенности / ограничения

- **Эта среда (где собирался код) не имеет доступа к внешним доменам кроме разрешённых** — поэтому прямой тест Mostbet/Telegram API отсюда невозможен, всегда даёт 403 `host_not_allowed`. Тестировать можно только на самом Railway.
- **Markdown в ответах Claude** — модели иногда норовят вставить `**жирный**` или `## заголовок`, несмотря на явный запрет в промпте. Если увидишь это в проде — надо усиливать формулировку запрета или добавлять пост-обработку (regex strip).
- **Многострочные f-строки** — при программном редактировании файла через скрипты несколько раз ловили `SyntaxError: unterminated string literal` из-за того что `\n` в строке заменялся на настоящий перенос строки. Если редактируешь переводы — всегда проверяй что `\n` остаётся буквенным эскейпом, не реальным переносом.
- **Модель `claude-sonnet-4-5-20251001` не существует** — в проекте когда-то была ошибка 404 на это имя. Актуальное имя — `claude-sonnet-4-6`.
- **python-telegram-bot 21.5 + конфликт инстансов** — если в Railway случайно запущено два деплоя одновременно, Telegram возвращает `409 Conflict` на `getUpdates`. Решение — убедиться что старый деплой остановлен перед новым.
- **`anthropic>=0.40.0` + `httpx>=0.27.0` обязательны** в `requirements.txt`, иначе `TypeError: proxies` при инициализации клиента.

## Что не реализовано / зоны роста

- Webhook поддержан в коде (`WEBHOOK_URL` переменная), но не протестирован в проде — пока бот живёт на polling
- Очередь запросов к Claude через `asyncio.Semaphore(5)` добавлена, но не нагрузочно тестировалась
- Статистика точности прогнозов считается только на основе ручного фидбека пользователя (✅/❌ в истории), нет автоматической сверки с реальным результатом матча
- Нет монетизации (платных подписок) — осознанно не делалось по запросу заказчика
